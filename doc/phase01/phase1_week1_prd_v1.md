# PRD v1（Phase 1 / Week 1）

## 1. 本周目标
在不扩展 Phase 2+ 范围的前提下，冻结 Phase 1 需求边界，为 Week 2 开发提供单一事实源。

## 2. 需求范围（In Scope）
1. 最小指令集执行语义冻结（可测试、可重放）。
2. 执行阶段仅产出 `state_diff`，禁止直接提交状态。
3. 未经 Verifier 的提交路径必须硬阻断。
4. 非确定性风险（时间/随机/未声明外部依赖）可扫描、可阻断。
5. 形成 `gate-report.json/.md` 的 Phase 1 门禁证据路径。

## 3. 非目标（Out of Scope）
1. RBAC/ABAC 全量上线（属于 Phase 2）。
2. Skill 生态签名与准入三件套（属于 Phase 4）。
3. 多节点验证协调与治理规模化（属于 Phase 5）。

## 4. 验收标准映射（Phase 1 Exit）
- 重放一致率 `>= 99.9%`
- 未验证提交率 `= 0`
- `result_hash` 覆盖率 `= 100%`
- 核心场景回归通过率 `>= 98%`

## 5. 用户故事（工程视角）
- 作为 Runtime 开发，我需要稳定的最小指令语义，以便实现可重放执行。
- 作为 QA，我需要可追溯证据与样本口径，以便判断是否达标。
- 作为门禁官，我需要机器可判定报告，以便做 Go/No-Go 决策。

## 6. 交接项
- 给架构师：接口契约必须覆盖 `ExecuteRequest/ExecutionResult/VerifyResult`。
- 给 QA：回放样本与对抗样本的最小集定义。
- 给项目经理：Week 2 必须按“执行器 -> 验证器 -> 阻断中间件”顺序排程。
