## 3. 性能指标定义

### 3.1 核心性能指标

| 指标 ID | 指标名 | 定义 | 目标值 | 优先级 |
|---|---|---|---|---|
| PERF-001 | P99 执行时延 | 99% 请求执行时延 | <300ms | P0 |
| PERF-002 | P99 验证时延 | 99% 请求验证时延 | <300ms | P0 |
| PERF-003 | 吞吐量 | 每秒处理请求数 | ≥100 请求/秒 | P1 |
| PERF-004 | 阻断开销 | 阻断中间件引入的开销 | <5% | P1 |
| PERF-005 | Batch 开销 | Batch 相比单条的开销 | <20% | P1 |

### 3.2 性能指标计算

**P99 执行时延**:
```
P99 = 第 99 百分位的执行时延
执行时延 = 请求接收时间 - 响应发送时间
```

**吞吐量**:
```
吞吐量 = 总请求数 / 总时间 (秒)
```

**阻断开销**:
```
阻断开销 = (有阻断时延 - 无阻断时延) / 无阻断时延 * 100%
```

**Batch 开销**:
```
Batch 开销 = (Batch 时延 - 单条时延) / 单条时延 * 100%
```

| 阻断开销 | >4% | >6% | 告警/阻断发布 |

### 7.2 CI/CD 集成

**性能回归检测流程**:
```yaml
# .github/workflows/performance-regression.yml

name: Performance Regression

on: [pull_request]

jobs:
  performance-test:
    runs-on: performance-cluster
    steps:
      - uses: actions/checkout@v3
      
      - name: Run Performance Test
        run: |
          k6 run scripts/performance_test.js
      
      - name: Check Regression
        run: |
          python scripts/check_regression.py \
            --p99-threshold 300 \
            --throughput-threshold 100
      
      - name: Upload Results
        uses: actions/upload-artifact@v3
        with:
          name: performance-results
          path: results/
```
