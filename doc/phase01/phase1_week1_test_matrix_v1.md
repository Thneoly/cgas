# TEST-MATRIX v1（Phase 1 / Week 1）

## 1. 测试目标
为 Phase 1 建立可执行的测试基线，覆盖功能正确性、确定性、一致性、阻断逻辑和回归门禁。

## 2. 测试矩阵

| 类别 | 用例示例 | 输入来源 | 预期结果 | Gate 映射 |
|---|---|---|---|---|
| 功能 | 最小指令执行成功路径 | 黄金样本（正常） | 输出合法 `state_diff` | 回归通过率 |
| 边界 | 空输入/极值参数/缺字段 | 边界样本 | 明确失败码，无脏提交 | 未验证提交率 |
| 对抗 | 时间/随机/未声明依赖注入 | 对抗样本 | 被扫描器阻断 | 确定性保障 |
| 一致性 | 同输入重复执行重放 | 黄金样本（核心） | hash 一致 | 重放一致率 |
| 链路 | Verifier 失败后提交尝试 | 构造样本 | 提交被拒绝并 REVERT | 未验证提交率=0 |
| 回归 | 核心场景批量回放 | 回放集 | 通过率达阈值 | 核心回归 >=98% |

## 3. 样本策略
- 核心样本量：`N >= 200`（正常/边界/对抗）
- 高危样本量：发布门禁目标 `N >= 1000`（不足仅允许修复发布）
- 抽检建议：`>= 20%`
