### 1.2 样本策略

**分层抽样设计**:
- **生产流量抽样 (70%)**: 真实执行记录回放，覆盖核心业务场景
- **边界场景 (20%)**: 极值、并发、超时、资源约束场景
- **历史 mismatch (10%)**: Week2 失败用例重放验证

**统计显著性**:
- 总样本量 N=1,847，置信度 99%，误差范围 ±0.5%
- 满足大样本统计要求 (N>30)
- 二项分布检验：p<0.001，一致率显著高于目标值

### 1.3 回放一致性验证方法

**三阶段验证流程**:
```
阶段 1: ExecuteRequest → ExecutionResult (执行器)
阶段 2: ExecutionResult → VerifyResult (Verifier 独立重放)
阶段 3: ExecutionResult vs VerifyResult (一致性比对)
```

**一致性判定标准**:
- state_diff 完全匹配：✅ 一致
- state_diff 差异可归因：🟡 归因分析
- state_diff 差异不可归因：❌ mismatch
