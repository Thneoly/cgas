# Week 2 SRE 计划（运维部署工程师）

## 1. 目标
- 提供稳定 staging 联调环境，支持开发与 QA 连续验证。
- 建立 Phase 1 发布前置检查项，提前验证回滚可行性。

## 2. 本周任务

| 任务ID | 任务 | 输出 | DoD |
|---|---|---|---|
| SRE-W2-1 | 联调环境健康检查 | 环境检查报告 | 核心服务可用且依赖正常 |
| SRE-W2-2 | 配置基线固化 | 配置清单 | 环境差异可追踪 |
| SRE-W2-3 | 初版回滚预案 | 回滚步骤文档 | 可在 staging 演练 |
| SRE-W2-4 | 关键 SLI 采集 | 周报数据 | 执行/验证路径可监控 |

## 3. 风险提示
- 若配置漂移未收敛，Week 3 重放一致性可能被环境差异污染。
- 若无回滚预案，Week 5 发布准备将出现集中风险。

## 4. DoD
- staging 联调阻塞数在周内持续下降。
- 回滚预案可执行并完成一次演练记录。
- 可向项目经理提供发布风险预评估结论。
