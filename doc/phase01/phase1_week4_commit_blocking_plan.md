# Week 4 开发计划（提交硬阻断）

## 1. 目标
确保任何提交动作都必须经过 Verifier，彻底封堵未验证提交路径。

## 2. 任务拆解

| 模块 | 任务 | 输入 | 输出 | DoD |
|---|---|---|---|---|
| Commit Middleware | 阻断中间件实现 | 提交链路清单 | 拦截与拒绝逻辑 | 未验证请求 100% 拦截 |
| Bypass Detector | 旁路检测规则 | Week3 安全审查 | 旁路检测报告 | 关键旁路全覆盖 |
| Error Contract | 阻断错误码标准化 | 现有错误码 | reasonCode 映射表 | QA 可稳定断言 |
| Audit Pipeline | 阻断事件审计 | trace/request 上下文 | 审计日志落库 | 门禁证据可导出 |

## 3. DoD
- 直提路径（包括隐式提交）全部被拒绝。
- 阻断事件具备 `status/reasonCode/evidence`。
- QA 可复现至少 1 组“阻断成功”与 1 组“验证后放行”场景。

## 4. 交接项
- 给 QA：阻断测试样例与断言规则
- 给观测：阻断事件字段与报表口径
- 给门禁官：阻断覆盖率证据包
