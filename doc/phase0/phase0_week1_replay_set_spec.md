## 1. 回放集目标

验证最小可运行链路 **Gateway → Executor → Verifier → Committer** 的确定性契约一致性。

### 1.1 契约字段(已冻结并签署)

| 字段 | 类型 | 说明 | 测试要点 | 安全审计 |
|------|------|------|----------|----------|
| `program` | string | 程序标识 | 唯一性、格式校验 | 5.1 可追溯 |
| `input` | object | 输入数据 | 序列化一致性、边界值 | 5.2 完整性 |
| `state_root` | string | 状态根哈希 | 确定性、可复现 | 5.3 防篡改 |
| `env_fingerprint` | object | 环境指纹(三层) | 主机/容器/平台、可追溯 | 5.1 可追溯 |

### 1.2 最小可运行链路

```
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│  Gateway  │───▶│  Executor │───▶│  Verifier │───▶│ Committer │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
     │                │                │                │
     ▼                ▼                ▼                ▼
  请求接收          执行结果         一致性验证        提交落账
  gRPC/REST        Rust serde       哈希比对          状态持久化
```

**接口契约** (Dev v2.0):
- 序列化: Rust serde (JSON + Protobuf)
- 协议: gRPC + REST双协议
- 超时: 默认30s，可配置

**失败路径与回滚机制** (Dev v2.0):

| 组件 | 失败场景 | 处理策略 | 回滚动作 | 测试用例 |
|------|----------|----------|----------|----------|
| Gateway | 请求超时 | 重试(最多3次) | 无 | A09 |
| Gateway | 协议错误 | 立即拒绝 | 无 | A06 |
| Executor | 执行异常 | 状态回滚 | 恢复前置state_root | B04 |
| Executor | 资源不足 | 排队等待 | 无 | A08 |
| Verifier | 一致性失败 | 标记不一致 | 通知人工介入 | N01-N05 |
| Committer | 提交失败 | 事务回滚 | 恢复前置状态 | A04 |

### 9.2 一致性判定规则

| 条件 | 判定 | 处置 |
|------|------|------|
| 相同input+state_root+env_fingerprint(三层) → 相同output | **Pass** | 记录成功 |
| 相同input+state_root+env_fingerprint(三层) → 不同output | **Fail** | 记录差异并调查根因 |
| env_fingerprint任一层次不一致 | **Invalid** | 标记为无效样本，不计入统计 |

### 9.3 SLO计算方法

```yaml
可用性: (成功请求数 / 总请求数) × 100%
  - 成功: HTTP 2xx/3xx
  - 失败: HTTP 4xx/5xx、超时、连接错误

延迟P99: 排序所有请求响应时间，取第99百分位
  - 目标: ≤100ms
  - 测量点: Gateway入口 → Committer出口

错误率: (错误请求数 / 总请求数) × 100%
  - 错误: 5xx响应、超时、系统异常
  - 目标: ≤0.1%

恢复时间: 故障发生到服务恢复的时间(秒)
  - 故障检测: 健康检查失败
  - 恢复确认: 连续3次健康检查成功
  - 目标: ≤5m (300秒)
```
