## 四、回放执行器技术方案 (Dev确认)

### 4.1 执行器架构

```
┌─────────────────────────────────────────────────────────┐
│            Playback Executor (Rust+并发)                 │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  Sample     │  │  Execution  │  │  Comparison │     │
│  │  Loader     │  │  Engine     │  │  Engine     │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
├─────────────────────────────────────────────────────────┤
│              Concurrent Execution (tokio)                │
├─────────────────────────────────────────────────────────┤
│                    Output: Pass/Fail + Metrics           │
└─────────────────────────────────────────────────────────┘
```

### 4.2 接口规范

| 接口 | 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|------|
| load_samples | GET | path, filter | [Sample] | 样本加载 |
| execute | POST | sample_id | ExecutionResult | 单次执行 |
| batch_execute | POST | [sample_id] | [ExecutionResult] | 批量执行 |
| compare | POST | expected, actual | CompareResult | 结果比对 |

### 4.3 性能目标

| 指标 | 目标值 | 验证方法 |
|------|--------|----------|
| 并发度 | ≥10 并行 | 压力测试 |
| 单样本执行延迟 | ≤50ms | 基准测试 |
| 200样本全量执行 | ≤30s | 端到端测试 |
| 内存占用 | ≤200MB | 资源监控 |

### 4.4 失败路径与恢复

| 失败模式 | 检测机制 | 恢复策略 |
|----------|----------|----------|
| 样本加载失败 | 校验和验证 | 重试3次→跳过记录 |
| 执行超时 | 超时监控(100ms) | 终止→记录→继续 |
| 比对失败 | 差异检测 | 记录差异→生成报告 |
