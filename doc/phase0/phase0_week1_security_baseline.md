## 7. 例外策略安全审查 (规则 4)

### 7.1 例外审批流程 (规则 4.1)

✅ Security 会签点已明确且全角色确认：

```
1. 申请 → 责任人提交例外申请 + 原因 + 补偿控制
2. 评审 → PM + Security + QA 联合评审 (Security 必须参与)
3. 批准 → 所有评审人签字批准 (Security 签字为必要条件)
4. 跟踪 → 例外项纳入风险台账，定期跟踪 (Security 复核)
```

### 7.2 例外有效期 (规则 4.2)

- 默认有效期：7 天 ✅
- 最长有效期：14 天 ✅
- 到期强制修复或重新申请 ✅

### 7.3 补偿控制 (规则 4.3)

Security 建议补偿控制措施：

| 场景 | 补偿控制 |
|------|----------|
| 自动化校验缺失 | 增加人工审查 |
| 样本量不足 | 提高抽检比例 (≥30% → ≥50%) |
| 高风险例外 | 限制发布范围 (灰度/金丝雀) |
