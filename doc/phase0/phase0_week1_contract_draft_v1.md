## 三、最小可运行链路定义

### 3.1 链路架构

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Gateway   │ →  │  Executor   │ →  │  Verifier   │ →  │  Committer  │
│  任务提交    │    │  契约执行    │    │  结果验证    │    │  状态提交    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 3.2 各节点职责与验收标准

| 节点 | 职责 | 输入 | 输出 | 验收标准 |
|------|------|------|------|----------|
| Gateway | 任务接收与路由 | HTTP/gRPC请求 | 任务ID + 队列确认 | 响应时间<100ms |
| Executor | 契约执行 | 任务ID + 契约 | 执行结果 + 证明 | 执行成功率>99% |
| Verifier | 结果验证 | 执行结果 + 证明 | 验证通过/失败 | 验证准确率100% |
| Committer | 状态提交 | 验证通过结果 | 状态根更新 | 提交延迟<1s |

### 3.3 SLO定义 (SRE确认)

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 可用性 | ≥99% | 健康检查点成功率 |
| 延迟 P99 | ≤100ms | 端到端链路延迟 |
| 错误率 | ≤0.1% | 失败请求占比 |
| 恢复时间 | ≤5m | MTTR从告警到恢复 |

### 3.4 健康检查点 (SRE定义)

| 节点 | 检查项 |
|------|--------|
| Gateway | HTTP 200 + 队列深度监控 |
| Executor | 任务执行成功率 + 资源使用率 |
| Verifier | 验证通过率 + 验证延迟 |
| Committer | 提交成功率 + 链上确认延迟 |
