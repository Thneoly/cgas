# Phase 0 提交闸门规则 v1.0

## 1. 目标

定义 Phase 0（基线冻结阶段）的门禁规则，确保：
- 确定性契约完整且可执行
- 回放集样本量达标且质量合格
- 门禁报告链路可正常产出

---

## 2. 核心规则

### 2.1 契约定义门禁（phase0_contract_defined）

**检查项**:
- 契约草案文档已完成
- 包含必填字段：`program`, `input`, `state_root`, `env_fingerprint`
- 版本策略明确

**判定**:
- ✅ 通过：文档完整且字段齐全
- ❌ 失败：缺少任一必填字段或版本策略

---

### 2.2 利益相关者审批（phase0_stakeholders_approved）

**检查项**:
- PM 签字确认
- Dev 签字确认
- QA 签字确认
- Security 签字确认（如适用）

**判定**:
- ✅ 通过：所有必须角色已签字
- ❌ 失败：任一角色未签字

---

### 2.3 契约冻结门禁（phase0_contract_frozen）

**检查项**:
- 契约冻结版文档已发布
- 争议项已收敛或降级为 backlog
- 变更流程已定义

**判定**:
- ✅ 通过：冻结版文档发布且变更流程明确
- ❌ 失败：仍有未收敛的高优先争议项

---

### 2.4 校验器就绪（phase0_validator_ready）

**检查项**:
- 契约校验器代码已完成
- 至少 1 个校验脚本可运行
- 字段覆盖率 = 100%

**判定**:
- ✅ 通过：校验器可运行且覆盖全部字段
- ❌ 失败：校验器不可运行或覆盖率 < 100%

---

### 2.5 回放集就绪（phase0_replay_set_ready）

**检查项**:
- 样本总量 >= 200
- 覆盖三类样本（正常/边界/对抗）
- 配比偏差 <= 10%

**判定**:
- ✅ 通过：样本量达标且分类完整
- ❌ 失败：样本量 < 200 或分类缺失

---

### 2.6 样本量门禁（phase0_sample_minimum）

**检查项**:
- 核心回放样本量 >= 200
- 高危样本量 >= 1000（如适用）
- 抽检通过率 >= 95%

**判定**:
- ✅ 通过：样本量达标且质量合格
- ❌ 失败：样本量不足或抽检不达标

---

### 2.7 门禁报告就绪（phase0_gate_report_ready）

**检查项**:
- gate-report.json 可正常产出
- Schema 校验通过率 = 100%
- 至少覆盖 1 组 PASS 和 1 组 FAIL 示例

**判定**:
- ✅ 通过：报告可产出且校验通过
- ❌ 失败：报告无法产出或校验失败

---

### 2.8 CI 集成门禁（phase0_ci_integration）

**检查项**:
- 门禁规则已集成到 CI 流程
- pre-merge / nightly / pre-release 触发器已配置
- 失败时自动创建工单

**判定**:
- ✅ 通过：CI 集成完成且触发器正常
- ❌ 失败：CI 未集成或触发器异常

---

## 3. 判定口径

### 3.1 统计窗口
- 日常监控：最近 7 天滚动窗口
- 发布门禁：最近 14 天窗口

### 3.2 最小样本量
- 功能回归（核心场景）：N >= 200
- 发布门禁（高危相关）：高危样本 N >= 1000

### 3.3 通过判定（三元约束）
指标必须同时满足：
1. 目标阈值
2. 分位约束（P95/P99）
3. 红线零违规

### 3.4 连续达标时长
用于阶段放行的红线指标，需在最近 14 天内连续通过 2 次双周关卡

---

## 4. 例外策略

### 4.1 例外审批流程

1. **申请**: 责任人提交例外申请，说明原因与补偿控制
2. **评审**: PM + Security + QA 联合评审
3. **批准**: 所有评审人签字批准
4. **跟踪**: 例外项纳入风险台账，定期跟踪

### 4.2 例外有效期
- 默认有效期：7 天
- 最长有效期：14 天
- 到期必须修复或重新申请

### 4.3 补偿控制
例外批准时必须定义补偿控制措施，例如：
- 增加人工审查
- 提高抽检比例
- 限制发布范围

---

## 5. 审计要求

### 5.1 必须记录的审计事件
- 门禁检查结果（通过/失败）
- 例外审批记录
- 指标数据来源
- 评审人签字

### 5.2 审计字段
每条审计记录必须包含：
- `event_id`: 事件唯一标识
- `event_type`: 事件类型
- `timestamp`: 时间戳
- `actor`: 操作者
- `evidence_refs`: 证据引用

### 5.3 审计存储
- 存储介质：不可变存储（WORM）
- 保留期限：至少 180 天
- 可追溯性：支持按 release_id 查询

---

## 6. 门禁触发后的处置

### 6.1 自动动作

| 门禁失败 | 自动动作 |
|----------|----------|
| phase0_contract_defined | 阻断进入 Week 2 |
| phase0_stakeholders_approved | 阻断契约冻结 |
| phase0_replay_set_ready | 阻断回放集构建 |
| phase0_sample_minimum | 阻断门禁链路打通 |
| phase0_gate_report_ready | 阻断 Phase 0 收官 |

### 6.2 恢复条件

| 门禁失败 | 恢复条件 |
|----------|----------|
| phase0_contract_defined | 补充缺失字段并重新评审 |
| phase0_stakeholders_approved | 完成所有角色签字 |
| phase0_replay_set_ready | 补充样本至 N>=200 |
| phase0_sample_minimum | 补充样本并重新抽检 |
| phase0_gate_report_ready | 修复报告生成逻辑 |
