## 1. 验证目标

验证**契约校验器**对冻结契约字段的正确执行，确保最小可运行链路(Gateway→Executor→Verifier→Committer)的契约一致性。

### 1.1 契约字段(已冻结，来自Week1 v2.0)

| 字段 | 类型 | 验证要点 | 用例数 | 争议状态 |
|------|------|----------|--------|----------|
| `program` | string | 唯一性、格式、版本兼容性 | 5 | ✅ 争议清零 |
| `input` | object | 序列化一致性、边界值、类型安全 | 5 | ✅ 争议清零 |
| `state_root` | string | 哈希匹配、确定性、防篡改 | 5 | ✅ 争议清零 |
| `env_fingerprint` | object | 三层结构(主机/容器/平台)、可追溯 | 5 | ✅ 争议清零 |
| **合计** | - | - | **20** | **3项争议清零** |

### 1.2 校验器架构(Dev v2.0)

```
┌─────────────────────────────────────────────────────────────┐
│                    契约校验器 (Rust)                        │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   Parser    │  │  Validator  │  │    Reporter         │  │
│  │  (serde)    │──│  (rules)    │──│  (evidence+audit)   │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
         │                   │                    │
         ▼                   ▼                    ▼
    gRPC/REST          规则引擎            证据库+审计日志
    (Dev v2.0)        (4字段×N规则)        (Week1规范延伸)
```

**接口规范** (Dev v2.0):
- 序列化: Rust serde (JSON + Protobuf)
- 协议: gRPC (内部) + REST (外部)
- 超时: 默认30s，可配置
- 错误码: 0=Success, 1-99=业务错误, 100-199=系统错误

**容器化部署** (SRE v1.0):
- 运行时: containerd
- 资源限制: CPU=2, Memory=4GB
- 安全边界: seccomp/apparmor (Week3)
- 网络策略: Calico (Week3)
