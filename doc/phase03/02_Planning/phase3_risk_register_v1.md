## 1. 风险台账概述

### 1.1 编制目的

本风险台账用于识别、评估和跟踪 Phase 3 执行过程中的各类风险，确保风险可见、可控、可收敛。

### 1.2 风险管理流程

```
风险识别 → 风险评估 → 缓解计划 → 持续跟踪 → 风险收敛
         ↓              ↓              ↓
       日检 (Agent)   周评 (PM)     Gate 评审 (门禁官)
```

### 1.3 风险等级定义

| 等级 | 影响 | 概率 | 响应要求 |
|---|---|---|---|
| **高 (H)** | 严重阻碍 Phase 3 目标达成 | >70% | 立即上报，每日跟踪 |
| **中 (M)** | 可能影响 Phase 3 进度/质量 | 30-70% | 周度评审，持续跟踪 |
| **低 (L)** | 轻微影响，可自行解决 | <30% | Agent 自行处理 |

### 1.4 风险收敛定义

| 收敛状态 | 定义 |
|---|---|
| 📋 未开始 | 缓解措施未启动 |
| 🔄 进行中 | 缓解措施执行中 |
| ✅ 已收敛 | 风险影响降至可接受水平 |
| ❌ 已发生 | 风险已触发，启动应急方案 |
| ⏸️ 已关闭 | 风险不再存在 (范围变更/外部因素) |
