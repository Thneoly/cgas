## 练习 7: 故障排查 ⭐⭐⭐

**时间**: 10 分钟  
**难度**: ⭐⭐⭐ 较难

### 场景

**假设**: 你发现仪表盘显示 NoData

### 任务

1. 诊断问题原因
2. 执行排查步骤
3. 记录排查过程

### 排查步骤

```bash
# 1. 检查 Prometheus targets
curl http://localhost:9093/api/v1/targets | jq '[.data.activeTargets[] | {health}]'

# 2. 检查是否有数据
curl 'http://localhost:9093/api/v1/query?query=up' | jq '.data.result | length'

# 3. 检查 Grafana 数据源
curl 'http://localhost:3003/api/datasources' -u admin:StagingGrafana2026 | jq '.[] | {name, type}'

# 4. 刷新浏览器 (Ctrl+Shift+R)
```

### 检查点

- [ ] 执行 targets 检查命令
- [ ] 执行数据检查命令
- [ ] 执行数据源检查命令

3. 执行修复操作
4. 验证修复效果
5. 记录事故报告

### 响应流程

```
1. 收到告警通知
   ↓
2. 登录 Grafana 查看告警
   ↓
3. 查看相关仪表盘定位问题
   ↓
4. 检查容器状态 (docker ps)
   ↓
5. 查看容器日志 (docker logs)
   ↓
6. 执行修复操作
   ↓
7. 验证修复效果
   ↓
8. 记录事故报告
```
