### 什么是 Grafana？

**Grafana 是**:
- 📊 可视化监控平台
- 🔔 告警管理工具
- 📈 数据分析平台

**我们的环境**:
| 环境 | URL | 用途 |
|------|-----|------|
| Alpha | http://localhost:3001 | 开发测试 |
| Beta | http://localhost:3002 | 集成测试 |
| Staging | http://localhost:3003 | 预发布监控 |

### Grafana 架构

```
┌─────────────┐
│   Grafana   │ ← 你在这里 (浏览器)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Prometheus  │ ← 数据存储
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Exporters  │ ← 数据采集
│ Node/Postgres│
│ Redis/Nginx │
└─────────────┘
```

## 第三部分：仪表盘使用 (10 分钟)

### 仪表盘概览

**Staging 环境 9 个仪表盘**:

| # | 仪表盘 | 用途 |
|---|--------|------|
| 1 | Staging Overview | 环境总览 ⭐ |
| 2 | System Overview | 系统资源 |
| 3 | Database Monitoring | 数据库 |
| 4 | Redis Monitoring | 缓存 |
| 5 | Nginx Monitoring | 负载均衡 |
| 6 | Application Metrics | 应用指标 |
| 7 | Alerts Overview | 告警 ⭐ |
| 8 | Prometheus Monitoring | 自监控 |
| 9 | Grafana Monitoring | 自监控 |

### 重点仪表盘详解

#### 1. Staging Environment Overview

**查看内容**:
- Prometheus Targets 状态
- CPU 使用率
- 内存使用率
- 应用健康状态
- 数据库健康状态

**关键指标**:
- Targets Up: 应该是 6 (全部健康)
- CPU Usage: 正常 <70%
- Memory Usage: 正常 <80%

#### 2. Alerts Overview

**查看内容**:
- 正在触发的告警
- 告警总数
- P0 告警数

**告警级别**:
- 🔴 **P0** - 严重 (立即响应)
- 🟡 **P1** - 警告 (尽快处理)

### 基本操作演示

#### 切换时间范围

**操作**:
1. 点击右上角时间选择器
2. 选择 `Last 1 hour`
3. 点击 `Apply`

**学员实践**: 切换到最近 1 小时

#### 刷新数据

**操作**:
1. 点击刷新按钮
2. 选择 `10s` (每 10 秒刷新)

**学员实践**: 设置自动刷新

#### 查看其他仪表盘

**操作**:
1. 点击左侧菜单 📊 **Dashboards**
2. 选择 **System Overview**
3. 查看系统资源监控

**学员实践**: 浏览所有 9 个仪表盘

## 第四部分：告警管理 (10 分钟)

### 告警规则

**P0 告警 (8 条)**:

| 告警 | 触发条件 | 响应时间 |
|------|---------|---------|
| ServiceDown | 服务宕机 1 分钟 | <5 分钟 |
| HighCPUUsage | CPU>85% 持续 5 分钟 | <5 分钟 |
| HighMemoryUsage | 内存>85% 持续 5 分钟 | <5 分钟 |
| DiskSpaceLow | 磁盘<15% 持续 5 分钟 | <5 分钟 |
| PostgresDown | 数据库宕机 1 分钟 | <5 分钟 |
| RedisDown | Redis 宕机 1 分钟 | <5 分钟 |
| HighErrorRate | 错误率>5% 持续 5 分钟 | <5 分钟 |
| ContainerRestarted | 容器 5 分钟内重启 | <5 分钟 |

**P1 告警 (16 条)**:

| 告警 | 触发条件 | 响应时间 |
|------|---------|---------|
| MediumCPUUsage | CPU>70% 持续 10 分钟 | <30 分钟 |
| MediumMemoryUsage | 内存>70% 持续 10 分钟 | <30 分钟 |
| ... | ... | <30 分钟 |

### 告警响应流程

**P0 告警响应**:

```
1. 收到告警通知 (5 分钟内)
   ↓
2. 查看告警详情
   ↓
3. 定位问题根源
   ↓
4. 执行修复操作
   ↓
5. 验证修复效果
   ↓
6. 记录事故报告
```

**P1 告警响应**:

```
1. 收到告警通知 (30 分钟内)
   ↓
2. 评估影响范围
   ↓
3. 安排时间修复
   ↓
4. 跟踪修复进度
```

### 告警查看演示

**操作**:
1. 点击左侧菜单 🔔 **Alerting**
2. 选择 **Alert rules**
3. 查看所有告警规则

**学员实践**: 查看告警规则列表

## 第六部分：Q&A + 评估 (10 分钟)

### 常见问题

**Q1: 仪表盘显示 NoData 怎么办？**

A: 
1. 检查时间范围是否正确
2. 检查 Prometheus targets 状态
3. 刷新浏览器 (Ctrl+Shift+R)

**Q2: 忘记密码怎么办？**

A: 
1. 联系管理员重置密码
2. 查看文档中的默认密码

**Q3: 如何添加新用户？**

A: 
1. 需要 Admin 权限
2. Configuration → Users → Add user
